Hands-On Power System Analysis with PyPowSyBl

Overview

Throughout this tutorial, we work with several publicly available test grids of varying size and complexity. This section introduces each one, loads it into PyPowSyBl, runs a quick power flow to verify it works, and provides an interactive single-line diagram for exploration.

These grids cover a range of real-world modelling scenarios — from a small didactic 6-bus network to a 6,500-bus French transmission grid — so you can observe how PyPowSyBl scales across different use cases.

This is a small synthetic network built into PowSyBl for testing purposes. It has 6 buses, 2 HVDC links, and multiple generators with active power control extensions. Its compact size makes it ideal for quick experiments and understanding basic concepts (slack distribution, HVDC modelling, electrical islands).

net_six_buses = get_six_buses()
lf_result = run_lf(net_six_buses)

ENTSO-E CGMES MicroGrid T4

The MicroGrid Test Configuration T4 is a standard test case published by ENTSO-E as part of the CGMES (Common Grid Model Exchange Standard) conformity assessment. It represents a small cross-border network with Belgium and Netherlands areas, including:

  • Multiple voltage levels (110 kV, 220 kV, 380 kV)
  • AC lines and transformers (2-winding and 3-winding)
  • Generators, loads, shunt compensators, SVCs
  • Phase and ratio tap changers
  • HVDC VSC converter stations

This is our primary test grid throughout the load flow tutorial — small enough to run instantly, yet rich enough to demonstrate all PowSyBl features.

net_microgrid_t4 = get_microgrid_t4()

print("Substation count:", len(net_microgrid_t4.get_substations()))
print("Voltage level count:", len(net_microgrid_t4.get_voltage_levels()))

ReliCapGrid Belgovia

ReliCapGrid is a realistic synthetic grid developed by the GridFM consortium for benchmarking reliability and capacity assessment tools. The Belgovia IGM (Individual Grid Model) represents a fictional country’s transmission network in the CIM-CGMES format, complete with boundary data (BDS).

This grid demonstrates PyPowSyBl’s ability to load CGMES Instance Files (XML) with separate boundary definitions — exactly how production-grade TSO data is exchanged in Europe via the ENTSO-E Operational Planning Data Environment (OPDE).

igm_belgovia = get_relicapgrid_belgovia()

print("Substations:", len(igm_belgovia.get_substations()))
print("Voltage levels:", len(igm_belgovia.get_voltage_levels()))

RTE 6515-Bus Network

This is a large-scale Matpower test case representing a snapshot of the French RTE (Réseau de Transport d’Électricité) transmission grid with 6,515 buses. It is loaded from a .mat file and includes phase tap changers that require a small fix (zeroing out the alpha at the current tap position) to avoid numerical issues.

We use this grid in the load flow tutorial to benchmark solver performance (Newton-Raphson vs Newton-Krylov) on a realistically sized network.

rte_6515 = get_rte_6515()

print("Substations:", len(rte_6515.get_substations()))
print("Voltage levels:", len(rte_6515.get_voltage_levels()))

lf_result = run_lf(
  rte_6515,
  use_reactive_limits=False,
  use_defaults=False,
  component_mode=lf.ComponentMode.MAIN_CONNECTED
)
print(lf_result)
